Overview

ID Card Maker creates membership ID cards as PNG images from member data.

  • Preview shows what the card will look like for the currently selected row.
  • Members table is where you enter or import member data.
  • Generate card(s) saves PNG files for selected rows.
  • Email card(s) sends emails (with cards attached) for selected rows.

Recommended workflow

  1. Set the template (required) and signature (optional). Use Set as default so they auto-load on startup.
  2. Add members by uploading a CSV or clicking Add member and typing directly into the table.
  3. Select rows you want to work on (checkboxes in the first column, or Select all in the header).
  4. Generate card(s) to create PNG files for the selected rows.
  5. Email card(s) to send cards to the selected rows’ email addresses.
  6. Open folder to view saved cards, and Export CSV to save your updated list.

Button availability: Buttons are disabled until prerequisites are met (for example, you must load a template before generating/emailing, and you must select at least one row).

Assets (Template & Signature)

The Template image is the background for the ID card and is required. The Signature image is optional and is overlaid on top of the template.

  • Choose file… selects an image from your computer.
  • Set as default saves your selection so it automatically loads at startup.
  • If you change template/signature later, the preview and newly generated cards use the latest selections.

Members table

Member data is entered and edited directly in the table.

Field What it’s used for Notes
Name Printed on the card and used in email templates. Use the full name.
ID Number Printed on the card; used as the base filename; required for generate/email. Format: 7 digits (example: 2010031).
Date Printed on the card. Required format: YYYY-MM-DD (example: 2026-04-20).
Email Destination for emailing cards. Required; must look like name@example.com.
  • Add member inserts a new empty row.
  • Remove row deletes a row using the × button at the far right of that row.
  • Select a row by clicking it to update the preview. (Selection is separate from the checkbox “work list”.)
  • Select work rows using the checkbox column; only checked rows are generated/emailed.

Buttons (order of operations)

The buttons above the members table are ordered to match the recommended workflow:

  1. Add member — inserts a new empty row.
  2. Generate card(s) — generates PNGs for the checked rows.
  3. Email card(s) — sends emails for the checked rows.
  4. Open folder — opens the current output folder.
  5. Export CSV — downloads a CSV containing the current table rows.

CSV import & export

Import (Upload members CSV)

The CSV must contain a header row with these columns (case/space-insensitive):

nameid_numberdateemail
Jane Smith20100312026-04-20jane@example.com
  • Dates are normalized to YYYY-MM-DD on import when possible. You can only generate/email when the Date column contains valid YYYY-MM-DD values.
  • After import, use the checkboxes to choose which rows to generate/email.

Export (Export CSV)

  • Exports the current table as CSV so you can archive it or re-import later.
  • Tip: Export after you’ve added rows or corrected emails/dates.

Generating card(s)

  • Load a Template image first.
  • Check the rows you want to generate, then click Generate card(s).
  • Cards are saved as PNG files (one per selected row), using ID Number as the base filename.
  • Rows missing required data may be skipped; check the “Batch” status for a summary.

Emailing card(s)

Email sending uses the configured active email account (Microsoft or Gmail).

  1. Open Email settings (top menu area).
  2. Choose the active account: Microsoft or Gmail.
  3. Optional: enable Union Management System and enter its email to send a copy of every card email.
  4. Enter Email and Password for the active account. (An app password may be required.)
  5. Optionally set From name, and customize the Subject and Body templates.
  6. Click Save. Then click Email card(s) above the members table.
  • Only checked rows are emailed.
  • Rows missing an ID Number or Email are skipped.
  • If a card image doesn’t exist yet, the app will generate it automatically (best-effort) as part of the email send.

Template placeholders

You can use these placeholders in Subject/Body templates:

  • {name}, {id_number}, {date}, {email}, {sender}

Default SMTP server settings

  • Microsoft: smtp.office365.com:587 (STARTTLS)
  • Gmail: smtp.gmail.com:587 (STARTTLS)
  • If your organization uses different SMTP settings, follow the instructions from your administrator.

Output folder (required)

The output folder is where generated PNG files are saved and where email attachments are found/created. Generating and emailing require an output folder.

  • Use Choose… to pick a folder, or type a path manually.
  • Open folder opens the current output folder in your file explorer.

Keyboard & tips

  • F1 — Open this Help page.
  • Preview vs selection: Clicking a row selects it for preview; checkboxes control what gets generated/emailed.
  • Status messages: Watch the “Batch” and “Email” status lines for summaries and errors.

Troubleshooting

  • Generate/Email buttons are disabled: Load a template and check at least one row.
  • Email says to enter email + password: Open Email settings, pick the active account, enter credentials, click Save, then try again.
  • Some rows were skipped: Ensure each checked row has an ID number and (for email) an email address.
  • Date didn’t print: Use YYYY-MM-DD. Unrecognized dates may be left blank.
  • Preview is blank: Load a template image, then click a row to select it.

Privacy & storage

  • Email credentials are used to send email. Depending on settings, passwords may be saved to support auto-loading on startup.
  • Generated cards and exported CSV files are saved on your computer.
